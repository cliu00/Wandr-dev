1. Recommendation Card Improvements

Each itinerary recommendation card currently repeats information about who selected the activity.

Example duplication:

“Alice’s Pick”

“For: Alice, Bob”

This is redundant.

Task

Choose the best and clearest format and remove the other.

Goal:

The card should clearly communicate who the recommendation is for without repeating information.

Example preferred format:

Alice’s Pick

or

Recommended for: Alice & Bob

But not both.

2. Add “Rest / Skip Activity” Option

Some travelers prefer downtime between activities.

Add an option in each recommendation card that allows the user to replace the activity with a rest block.

Example interaction:

Tap option:

“Swap for Rest”

Result:

Activity block becomes:

Free Time / Rest

Example description:

“Take a break, explore at your own pace, or relax at a café.”

This gives users flexibility without forcing a packed itinerary.

3. Multi-Day Navigation

Currently, itineraries for 3–4 day trips appear as one long scroll, which can be difficult to navigate.

Task

Add a clear visual cue that the itinerary is divided into days.

Possible UI patterns:

Day tabs (Day 1 / Day 2 / Day 3)

Sticky day navigation

Horizontal day selector

Goal:

Users should easily understand that they can switch between days.

Avoid endless scrolling.

4. Image Performance Optimization

Recommendation cards currently use large images.

Evaluate whether using smaller thumbnails could improve performance by reducing load size.

Questions to consider:

Does image size significantly impact load time?

Would smaller thumbnails reduce API/image loading costs?

Would image compression be a better solution?

If helpful, switch to smaller thumbnails while maintaining visual quality.

The goal is to reduce performance load without sacrificing visual appeal.

5. Save Trip – Account Requirement

Saving a trip should require a user account.

Task

When a user taps Save:

If the user is not logged in, prompt them to:

Log in

Create an account

This should appear as a simple authentication prompt rather than a disruptive screen.

Goal:

Encourage account creation without breaking the flow.

6. End of Itinerary – Add a Clear Next Step

Currently the itinerary page ends without a clear next action.

This creates a dead-end experience.

Task

Add a section at the end of the itinerary that proposes clear next steps.

Possible actions:

Save Trip

Share Trip

Invite Others

Regenerate Itinerary

Adjust Preferences

Goal:

Ensure users always know what they can do next.

Add a simple way to collect:

number of participants

each person’s name

This ensures that when group members submit their responses, the system can show:

who has completed the survey

who has not

Example flow:

User enters:

Alex
Jamie
Sam

The system then sends individual survey links associated with each name.

8. MVP Sharing Method

For the MVP:

Use Email Only for sharing invites.

Remove other sharing methods such as:

SMS

social links

messaging apps

This keeps the MVP simpler.
