Prototype Refinement + UX Audit

You are a Senior Product Manager, UX Designer, and Full-Stack Engineer helping improve an existing travel app prototype.

Important Rule

Do NOT redesign the application.

Edit and refine the existing prototype by improving:

UX flow

interaction logic

copy clarity

group collaboration features

quiz structure

usability

Maintain the current design system and layout as much as possible.

1. Quiz Improvements
Combine Trip Length + Dates

Currently there are separate questions for:

trip length

trip dates

Combine these into one question.

Example structure:

How long is your trip?

Options:

1 day
2 days
3 days
4 days

Optional field below:

Add travel dates (optional)

Explain internally that dates help with:

weather suggestions

seasonal tips

events

This reduces friction and keeps the quiz fast.

2. Group Trip Logic Improvements

Current rule:

The first user submission generates the initial itinerary.

As additional group members submit preferences:

the itinerary dynamically updates

group preferences influence the final itinerary.

Improve the UX so this behavior is clear and transparent to users.

Add Two Tabs to the Group Results Page

Add a simple tab layout:

Tab 1 — Current Itinerary

Shows the latest generated itinerary based on submitted preferences.

This is the working group plan.

Tab 2 — Group Preferences

Show a summary of each participant’s selections.

Example:

Alex
• Relaxed pace
• Food & markets
• Photography

Jamie
• Balanced pace
• Culture & museums

Sam
• Packed pace
• Nightlife

This transparency helps groups understand why the itinerary looks the way it does.

3. Group Participation Features
Reminder Feature

Add the ability to send a reminder to group members who have not submitted their preferences.

Example UX:

If someone has not completed the intake:

Display:

“Waiting for 2 people to add their preferences.”

Add button:

Send Reminder

Possible reminder methods:

resend link

notification

copy invite link again

For each flow:
